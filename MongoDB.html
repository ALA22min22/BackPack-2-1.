MongoDB Interview Questions & Answers

Role: IT Company Hiring Manager (HR)

CRUD Operations (5 Questions)

Q1: What is CRUD in MongoDB?

CRUD stands for Create, Read, Update, and Delete. These are the basic operations used to manage data in MongoDB collections.

Q2: How do you insert data in MongoDB?

Data is inserted using insertOne() or insertMany() methods. These methods add documents into a collection.

Q3: How do you read data from MongoDB?

Data is read using find() or findOne() methods. They retrieve documents based on given conditions.

Q4: How do you update documents in MongoDB?

Documents are updated using updateOne(), updateMany(), or replaceOne(). These methods modify existing data.

Q5: How do you delete documents in MongoDB?

Documents are deleted using deleteOne() or deleteMany(). These methods remove data from a collection.

Querying (5 Questions)

Q6: What is a query in MongoDB?

A query is used to retrieve specific data from a collection. It works like a filter based on conditions.

Q7: What is the use of find() method?

The find() method retrieves multiple documents that match a given condition from a collection.

Q8: What are query operators?

Query operators are special keywords like $gt, $lt, and $in. They help perform advanced filtering.

Q9: What is projection in MongoDB?

Projection is used to include or exclude specific fields in query results. It improves performance.

Q10: What is sorting in MongoDB?

Sorting arranges query results in ascending or descending order using the sort() method.

Integration with Node.js (5 Questions)

Q11: How does MongoDB connect with Node.js?

MongoDB connects with Node.js using drivers or libraries like Mongoose or the MongoDB native driver.

Q12: What is Mongoose?

Mongoose is an ODM library for MongoDB in Node.js. It provides schema-based data modeling.

Q13: Why are schemas important in Mongoose?

Schemas define the structure of documents. They help maintain data consistency and validation.

Q14: How are asynchronous operations handled?

MongoDB operations in Node.js are asynchronous and handled using callbacks, promises, or async/await.

Q15: What is a model in Mongoose?

A model is a wrapper around a schema. It is used to interact with the database collection.

Indexing in MongoDB (5 Questions)

Q16: What is an index in MongoDB?

An index improves query performance by allowing MongoDB to search data faster.

Q17: Why are indexes important?

Indexes reduce query execution time. Without indexes, MongoDB scans the entire collection.

Q18: What types of indexes are available?

Common indexes include single-field, compound, text, and unique indexes.

Q19: What is a compound index?

A compound index is created on multiple fields. It helps optimize queries using those fields together.

Q20: Can indexes affect performance negatively?

Yes, too many indexes can slow down insert and update operations because indexes also need updating.
