Basic Server (5 Questions)

Q1: What is Node.js?

Q2: What is a server in Node.js?

A server in Node.js listens for client requests and sends responses. It is commonly created using the http module.

Q3: How do you create a basic server in Node.js?

A basic server is created using the http.createServer() method, which handles incoming requests.

Q4: What is the role of the http module?

The http module allows Node.js to create web servers and handle HTTP requests and responses.

Q5: What is the difference between Node.js and a traditional server?

Node.js uses non-blocking, event-driven architecture, while traditional servers often use blocking threads.

File System (5 Questions)

Q6: What is the fs module in Node.js?

The fs module is used to work with the file system. It allows reading, writing, updating, and deleting files.

Q7: Difference between synchronous and asynchronous file operations?

Synchronous operations block code execution, while asynchronous operations do not block and improve performance.

Q8: How do you read a file in Node.js?

Files can be read using readFile() for async or readFileSync() for synchronous operations.

Q9: How do you write data to a file?

Data is written using writeFile() or appendFile() methods provided by the fs module.

Q10: Why is asynchronous file handling preferred?

It prevents blocking the main thread, allowing better performance and responsiveness.

Middleware (5 Questions)

Q11: What is middleware in Node.js?

Middleware is a function that runs between request and response. It modifies or processes requests.

Q12: Where is middleware commonly used?

Middleware is widely used in Express.js for logging, authentication, and request validation.

Q13: What are common types of middleware?

Common middleware types include application-level, router-level, error-handling, and built-in middleware.

Q14: What is next() in middleware?

The next() function passes control to the next middleware in the request-response cycle.

Q15: Why is middleware important?

Middleware helps organize code, reuse logic, and manage requests efficiently.

Single Threaded Application (5 Questions)

Q16: Is Node.js single-threaded?

Yes, Node.js uses a single-threaded event loop to handle multiple requests efficiently.

Q17: How does Node.js handle multiple requests?

Node.js uses an event-driven, non-blocking model that allows handling many requests concurrently.

Q18: What is the event loop?

The event loop manages asynchronous operations and executes callbacks when tasks are completed.

Q19: What are the advantages of single-threaded architecture?

It uses less memory, avoids thread management issues, and is highly scalable.

Q20: What are the limitations of single-threaded Node.js?

CPU-intensive tasks can block the event loop and reduce performance.
