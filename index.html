React JS Interview Questions & Answers

Role: IT Company Hiring Manager (HR)

React Lifecycle (6 Questions)

Q1: What is the React component lifecycle?

A React lifecycle is the series of phases a component goes through, such as mounting, updating, and unmounting.

Q2: What happens in the mounting phase?

In the mounting phase, the component is created and inserted into the DOM for the first time. During this phase, initial rendering happens and setup tasks can be performed.

Q3: What is componentDidMount used for?

It is used to perform side effects like API calls after the component is rendered.

Q4: What is the updating phase?

The updating phase occurs when props or state change and the component re-renders.

Q5: What is componentWillUnmount?

It is called before a component is removed from the DOM and is used for cleanup.

Q6: How are lifecycle methods handled in functional components?

They are handled using the useEffect hook instead of class-based lifecycle methods.

Routing (6 Questions)

Q7: What is routing in React?

Routing allows navigation between different pages or components without reloading the page.

Q8: Which library is commonly used for routing?

React Router is the most commonly used library for routing in React applications.

Q9: What is BrowserRouter?

BrowserRouter uses the browser history API to keep the UI in sync with the URL.

Q10: What is the Route component?

Route is used to render a component when the URL path matches a specific route.

Q11: What is Link used for?

Link is used to navigate between routes without reloading the page.

Q12: What is protected routing?

Protected routing restricts access to certain routes based on authentication status.

State Management (6 Questions)

Q13: What is state in React?

State is an object that stores dynamic data and controls component behavior.

Q14: Difference between state and props?

State is managed inside the component, while props are passed from parent to child.

Q15: How do you update state?

State is updated using the setState method or state updater functions.

Q16: What is lifting state up?

Lifting state up means moving state to a common parent component to share data.

Q17: What is useState hook?

useState is a hook that allows functional components to manage state.

Q18: Why should state not be mutated directly?

Direct mutation can cause unexpected behavior and prevent proper re-rendering.

Forms in React (6 Questions)

Q19: How are forms handled in React?

Forms are handled using controlled components where form data is stored in state.

Q20: What is a controlled component?

A controlled component is an input element whose value is controlled by React state.

Q21: What is an uncontrolled component?

An uncontrolled component manages its own state using the DOM.

Q22: How do you handle form submission?

By using an onSubmit handler and preventing the default browser behavior.

Q23: How do you handle multiple inputs?

By using the name attribute and a single change handler.

Q24: Why is validation important in forms?

Validation ensures correct and secure data is submitted by the user.

React Authentication (6 Questions)

Q25: What is authentication in React?

Authentication verifies the identity of a user before allowing access.

Q26: How is authentication commonly implemented?

It is commonly implemented using JWT tokens or session-based authentication.

Q27: What is JWT?

JWT (JSON Web Token) is a secure token used to transmit user information.

Q28: Where should tokens be stored?

Tokens are usually stored in HTTP-only cookies or localStorage.

Q29: What is role-based authentication?

It restricts access to features based on user roles like admin or user.

Q30: How do you protect routes in React?

By using protected routes and checking authentication before rendering components.
